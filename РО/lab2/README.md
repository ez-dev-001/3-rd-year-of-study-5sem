# Lab 02 — Parallel Matrix–Matrix Multiplication (MPI, Fox Algorithm)

## Мета роботи:
Метою лабораторної є ознайомлення з принципами **паралельного множення матриць** за допомогою бібліотеки **MPI** (Message Passing Interface) та алгоритму **Fox**.  
У ході роботи реалізовано **послідовну** та **паралельну** версії множення квадратних матриць, проведено експериментальні вимірювання часу виконання, обчислено **прискорення (Speedup)** та проведено аналіз ефективності.

---

## Опис реалізації

### Послідовний алгоритм
У послідовному варіанті використовується стандартне множення матриць за формулою:

$$
C_{ij} = \sum_{k=1}^{n} A_{ik} \cdot B_{kj}
$$

Кожен елемент результату обчислюється як скалярний добуток i-го рядка матриці A та j-го стовпця матриці B.  
Часова складність алгоритму — **O(n³)**.  


---

### Паралельний алгоритм (Fox Algorithm)

Алгоритм **Fox** базується на **блоковій декомпозиції** матриць та **комунікації між процесами** у вигляді двовимірної сітки.

#### Основна ідея:
1. Матриці `A` та `B` розбиваються на **q×q блоків**, де `q = √p`, а `p` — кількість процесів.  
2. Кожен процес отримує свій блок `Aij` та `Bij`.  
3. На кожній ітерації:
   - Ведучий процес у рядку розсилає свій блок **A** усім процесам цього рядка (`MPI_Bcast`).
   - Кожен процес виконує **локальне множення** отриманого блоку A з власним блоком B.
   - Потім блоки **B циклічно зсуваються вгору** по колонках (`MPI_Sendrecv_replace`).
4. Після `q` ітерацій кожен процес має свою частину результату `Cij`.

#### Використані комунікатори:
- **GridComm** — комунікатор усієї двовимірної сітки процесів.  
- **RowComm** — для обміну даними по рядках.  
- **ColComm** — для обміну даними по колонках.

## Аналіз результатів
При малих розмірах матриць паралельна версія не показує вищої ефективності через накладні витрати на комунікацію. При **збільшенні розміру задачі** видно помітне прискорення (Speedup ≈ 2–3). Загалом, із ростом розміру матриці ефективність паралельної версії зростає, і Fox algorithm показує хорошу масштабованість для великих задач.
